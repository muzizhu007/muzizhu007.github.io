Getting Started with iBeacon

发表于 2016-04-01   |  

本文为翻译Apple Developer开发文档Getting Started with iBeacon，由于本人水平问题，肯定存在诸多问题，请见谅！

概况

iBeacon是在iOS7新推出的感知位置可能性的应用。利用低功耗蓝牙（BLE），携带iBeancon技术的设备能够围绕某一对象建立监测区域。依据iOS设备与信号源之间的距离，能够判断设备何时进入或者离开该区域。所以在使用iBeacon技术时需要有硬件和软件两方面的考虑，本文档对两方面都进行了介绍，包括使用建议和最佳实践，确保有个高效的部署，使得有个优秀的用户体验。

iBeacon有三个不同的角色，根据你的需求，你有可能扮演其中的一个，两个甚至三个：

1.    APP 开发者
    如果你想要添加iBeacon到你的应用内，你需要使用Core Location APIs ,当iOS设备进入或者离开检测区域时，能够收到iOS的通知。你同样也能够将设备发送iBeacon广告。所以这些你只需在程序内包含iOS SDK即可，无需license。

2.    部署iBeacon设备
    无论你管理一个体育竞技场，博物馆，零售商店，或其他任何地方来使用beacons的设备，你需要了解这些设备的工作原理，周围的信号强度和材料的问题，并了解如何校准和测试部署。如果你想使用iBeacon Logo在会场内，而不会使用iBeacon技术的设备，在使用之前需要申请iBeacon Logo 的licese。可以通过访问
    https:// developer.apple.com/ibeacon/ 
    申请。

3.    使用iBeacon技术制造设备
    如果你有兴趣制造iBeacon技术的设备，你需要在制造之前申请license。可以通过访问https://developer.apple.com/ibeacon/ 
    申请iBeacon license。 被许可方可以访问相关技术规范，使用iBeacon logo以及iBeacon的识别指南。

iBeacon技术设备

iBeacon技术的设备可以使用纽扣电池供电一个月以上，或者使用更大的电池提供更长时间，或者使用外部电源更长时间工作。iOS设备统一能够配置生成iBeacon广告，但是功能范围上受限。可以使用在销售点或者售货亭，或者只是短时间使用内使用iBeacon的应用。

iBeacon软件 － Core Location APIs

iOS7之前的版本Core Location 使用“电子围栏”技术（经纬度和半径）定义监测区域。iBeacon 则提供了新的更灵活的方式定义一个监测区域。这个允许信源绑定对象不被固定在一个地方。例如：信标装置可以用于设定一个区域的周围像食物卡车或游轮上的可移动对象。此外，相同的标识符可以由多个设备。这将允许一个零售连锁店在所有分店使用相同标识符，并允许iOS设备知道当它进入其中的任何一个。

iBeacon精度

为保证高效的用户体验，如何考虑信号从信源发出后被检测到和确定精度是非常重要的。当iOS设备接收到信源的信号时，它使用信号强度（RSSI, or Received Signal Strength Indication）来确定接近信源，即接近其预计的精度。信号越强，iOS越接近信源，信号越弱，iOS越远离信源。

可以从现有iOS设备的GPS的工作上理解精度的概念。当iOS设备能够清晰的接受到GPS信号时（比如在开放的户外环境），你的位置就能更精确的被确定。在地图应用中可以很容易的看出定位精度，有一个蓝色的圆圈表示。如果设备在室内或者GPS信号被阻挡，那么蓝色圆圈放大表示精度降低。如果信号到卫星的阻挡减少（设备移动到户外或从背包拿出）蓝色圈则会变小。更好的信号接收设备能够缩小误差的幅度，更准确定位。

iBeacon的信号被设备接收，信号强度通常与设备到信号源的距离相关。在理想条件下（设备和信号源之间线路畅通），越靠近结果越精确。
如Figure 1显示，设备原理信号源时，信号强度越弱。由于信号减弱，iOS设备就不会有高可信度的精度表示到信号源的接近距离。类似于上面GPS中较大的蓝色圆圈。

当设备越靠近信号源，接收到的信号就增大，并且精度也提高。这就类似于GPS的蓝色圆圈变小。如Figure 2 所示，当设备越靠近信号源时精度的可信度就越高。

然而，和GPS类似，信号强度会被物体阻挡减少，如建筑物，或放置在背包，钱包，口袋里。信号衰减，或信号的强度的损失，可以由许多因素引起。信号源周边的物体，如Figure 3 显示有墙壁在信号源和设备之间，就会影响接收信号强度，这有可能导致设备认为信号源还在比它实际位置更远的地方。

人体本身也会使蓝牙信号衰减。当你背对信号源时（身体处于设备和信号源之间）就会影响信号强度，减少精度。Figure 4显示信号强度被减弱当人处于信号源和设备之间时。

当创建应用使用GPS或者iBeancon时，考虑精度是非常重要的。从Core Location对象获取的值(CLLocation类horizontalAccuracy属性，或者CLBeacon类的accuracy 属性)表明其不确定性或误差。两者的单位都是米。该值越高，位置的准确性越低。请记住取决于物理环境中的低准确度可能是不可能的。

区域监控

类似于现有的“电子栅栏”的区域监控，当设备进入或者离开一个iBeacon监控区域时，应用能够接收到通知信息。当一个应用使用这个请求开始监控一个Beacon区域时，它必须指定iBeacon广告的UUID。 当一个app被限制在20个监控区域时，使用单一的UUID在多个地址，一个设备能够容易的同时监控多个物理地址。如上连锁零售店的例子，一个设备能够监控3个不同的店而使用同一个UUID。这个UUID为基础的方法相比，“电子栅栏”的影响不可低估：用一行代码的应用程序可以建立周围的物体或地点任意数量的监视区。

除了UUID，应用程序还可以选择Major和Minor字段进一步指定一个Beacon监控区域。同样是连锁零售店例子，如果app只指定UUID，在进入或离开任何一家监控的分店时，都会收到通知。由于Major字段被用来指定特定分店，app会使用UUID＋Major配置Beacon监控区域。或者用户只对特定的部门感兴趣，则app使用UUID+Major+Minor的值。这种颗粒级别由程序开发者设定，并能够动态的指定。

现有的区域检测，用户进入或者推出时都会收到通知。如果app没有实时运行（比如由于内存关系被关闭），然后app会在后台运行并发生通知。一个重要的考虑是在iOS7设备中，如果用户明确后台程序不刷新（不论是全局或者指定你的app），然后你的app就不会再收到区域监控的通知。但是他还能继续使用测距的API。

由于基于低功耗蓝牙技术，典型的范围在几十米，比“电子栅栏”（百米至少）更加精确。通过上述论述，“电子栅栏”在室内精度较低，所以使用iBeacon技术能极大的提高室内的区域监控。然而，精确度又会被Beacon打物理位置所影响，不论用户将设备放在背包里，或者简单的处于设备和beacon之间都会影响监控区域的进入和退出检测。

iBeacon用户体验的考虑

虽然有接近状态和准确性之间的相关性，但是映射关系不一定是1:1。考虑连锁零售店安装遍布全国的例子，一个应用程序可以使用区域的监测，检测进入到店里来触发本地通知，欢迎用户到店，并邀请他们来启动应用程序。未避免骚扰到用户，app可以设置为只有用户第一次进入商店时，发送通知信息。

一旦内嵌该应用，一个店内定制接口可以提交。如果iBeacon的广告Major字段包含特殊分店地址，那么app就能立马知道用户进入哪个商店。一个理想的测距情况是应用程序运行在最前端，设备拿在用户手上，屏幕点亮。大型家装卖场往往有许多走道和部门。将Beacon设备安装在每个通道的端部和各部门内，app就能够通过和Beacon的接近状态，可以在地图应用中显示用户的大致位置。

虽然在这个例子中有许多情况导致出现Near或Immediate（如果用户手持iphone到特定的Beacon点时），但是由于物理对象（通常为金属架，大型笨重显示项目等），所以客户却看到出现proximities或者Far的情况。在这种情况下，app可能会显示附近的其他Beacon点，而不会锁定到用户附近的信号源。相反，app可能希望用户手动指定该最有关联的店（不论是他们的兴趣或者他们实际最近的Beacon位置）。

优惠券

优惠券也能够通过iBeacon技术的设备传递出去。通过包含信号的UUID，优惠券信息能够发送当进入beacon监控区域。这个工作与（“电子栅栏”）指定latitude 和longitude的在位置你经过的方式相同。你能够指定UUID，或者可选的Major，Minor字段值组成的组合在Beacon区域内，能够发送信息。

部署iBeacon

当你准备部署基于iBeacon技术的任何实现时，你需要仔细评估你的解决方案的真实世界中的性能。

物理限制

iBeacon 设备使用低功耗蓝牙传输广播信号。BLE基于2.4G频率，因此在穿过各种物体时，例如墙，门或其他物体结构时，都会衰减。2.4G频率信号同样会被水影响，所以人体也会减少信号强度。这是非常重要的需要了解的，iOS设备接收到的信号会被衰减，被减弱。综上所如，当接受到的信号减弱后，iOS设备预估到iBeacon的接近程度的能力也会降低。

校准iBeacon

为提供最佳的用户体验，在部署环境中执行校准是关键的。每一个Beacon的安装，您都需要执行校准。Core Location使用1米距离校准估算模型。执行这个校准你需要：

1.    安装Beacon，并确保发出信号
2.    使用iOS7以上iPhone或ipod touch，并开启蓝牙，反复在1米位置上采样信号强度至少10s以上。当获取这些读数时，您需要手持设备垂直于设备上半部分。
3.    缓慢的移动设备在30cm线，保持方向，并且等距的测量设备如图所示。
4.    在校准的期间，收集CLBeacon的rssi属性的值
5.    平均收集的RSSI值，以获得测得的值。
6.    应用这个测量值，参考使用的Becean的详细说明，不同厂商之间有可能不相同

如上所讨论的，物理环境可以影响信号强度。因为每个Beacon的安装环境几乎可以肯定会发生变化，所以重复以上步骤安装Beacon是非常重要的。

最佳实践

为了获得最佳的用户体验和成功部署，一定要考虑以下最佳做法：

1.    测距的API不会在后台使用。为获得最佳结果，测距时应该是你的App在最前端，用户与您的app进行交互
2.    当使用测距API时，多个iBeacon设备被检测到，Core Location会上报这些Beacons顺序，是离其最近的最佳的预估。由于上面所讨论的信号衰减的问题，这个顺序可能是不正确的。例如，如果两个信标被附加到对象，并通过iOS设备被检测到，但一个信号是比另一个相当强，但在物理上是远，这可能导致远离beacon被首次报道。App需要仔细检测Beacon上报的Proximity区域，如果所有上报的都处于Far区域，再考虑上报的这两个对象是在附近，并且可以提供用户选择感兴趣的对象。
3.     利用位置授权提示框可选的文字来解释App要求使用用户位置的原因。如果你的app出现在屏幕上，解释为何用户同意使用该App了解他们位置的好处。您可以使用可选的文字使用NSLocationUsageDescription 在你的app的.plist文件中。
4.    如果你是购买iBeacon技术的第三方设备，了解这些设备如何进行配置和谁去做好安装，维护保养等方面的是非常重要的。
5.    当部署beacon设备在场所内时，一定要确保训练需要与它们交互的员工。例如，如果你正在部署零售解决方案，确保您的零售销售人员进行培训，对iOS应用的设备，什么样的好处是给你的客户的交互方式，支持iOS设备型号，故障排除建议，等等。
6.     如果您计划在指示牌的位置，这是鼓励你有iBeacon商标和标识许可。请访问<https://developer.apple.com/ibeacon> 申请iBeacon License。
